La définition générale (Ce qu'est CMake) :

"CMake n'est pas un compilateur. C'est un 'générateur de configuration'. Son rôle est de lire ce fichier texte simple, de comprendre l'architecture de mon projet, et de générer automatiquement le fichier Makefile adapté à mon ordinateur (mon WSL). Ensuite, c'est le vrai compilateur (comme GCC ou Clang) qui lira ce Makefile pour transformer le code en exécutable."

Bloc 1 : cmake_minimum_required et project

"La première ligne est une sécurité. Elle exige qu'une version récente de CMake soit installée sur la machine pour éviter des bugs de rétrocompatibilité. La seconde ligne définit le nom de mon projet, qui deviendra le nom du fichier exécutable final."

Bloc 2 : set(CMAKE_CXX_STANDARD 17)

"Le langage C++ évolue en permanence. Ici, j'impose à mon compilateur d'utiliser la norme C++17 (sortie en 2017). C'est le standard industriel actuel, qui offre des fonctionnalités modernes et sécurisées. Le paramètre REQUIRED True signifie que si le compilateur de l'ordinateur est trop vieux pour le C++17, le processus s'arrête net avec une erreur claire, plutôt que de compiler un code buggé."

Bloc 3 : L'astuce HPC (-Wall -Wextra -O3)
C'est ici que tu vas faire la différence.

"Cette ligne configure les 'flags' (drapeaux) de mon compilateur :

Les drapeaux -Wall et -Wextra demandent au compilateur d'afficher absolument tous les avertissements (Warnings) s'il détecte des choses bizarres dans mon code, même si ça ne le fait pas crasher. C'est pour garantir un code propre.

Le drapeau -O3 (Optimisation niveau 3) est fondamental pour le calcul scientifique. Il ordonne au compilateur de réécrire silencieusement mon code machine pour qu'il s'exécute le plus vite possible (par exemple, en déroulant les boucles for ou en utilisant les registres vectoriels du processeur). Sans ce -O3, l'assemblage de mes matrices prendrait beaucoup plus de temps."

Bloc 4 : L'inclusion d'Eigen (include_directories)

"C'est l'étape la plus importante pour utiliser Eigen. Je définis le chemin absolu vers le dossier d'Eigen sur mon système Linux. La particularité d'Eigen, c'est que c'est une bibliothèque 'Header-Only' (composée uniquement de fichiers .hpp). Je n'ai donc pas besoin de la compiler au préalable ni de la 'linker' (relier). Il me suffit de dire au compilateur, via include_directories, où aller lire les définitions mathématiques quand je fais un #include <Eigen/Sparse> dans mon code."

Bloc 5 : add_executable

"C'est la conclusion logique. Cette commande ordonne de fabriquer l'exécutable final (nommé Four_Thermique) en compilant et en fusionnant ensemble mon fichier principal (main.cpp) et mon fichier contenant la logique mathématique (utils_mef.cpp). Si je rajoute d'autres fichiers .cpp à l'avenir, c'est uniquement ici que je devrai les déclarer
